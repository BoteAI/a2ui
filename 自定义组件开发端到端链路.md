## 开发

### 目录约定
博特平台提供了一个脚手架用于开发a2ui自定义组件
git地址：  https://git-nj.iwhalecloud.com/yanxi/bote-remote-comp.git 

在 `src/pages/a2ui-remote` 下面新建一个目录，以组件命名。

**禁止**：与官方 basic catalog 组件同名（`Text`、`Button`、`Column` 等）。

#### 简单目录（示例：RemoteReactBadge、RemoteGlowCapsule）

```
src/pages/a2ui-remote/RemoteReactBadge
  api.ts          # 协议 API（defineComponentApi）
  element.tsx     # 组件实现（原生 createNativeElement 或 createReactComponent）
  index.ts        # 导出 mergeRegistryEntries
  index.less      # 组件样式（可选，Shadow 内注入）
```

#### 推荐目录（复杂 UI：antd、表单、级联等）

将 **标准 React UI** 与 **A2UI 桥接** 拆开，便于单测、Storybook 和在非 A2UI 场景复用。示例：`RemoteRegionCascade`。

```
src/pages/a2ui-remote/RemoteRegionCascade
  api.ts              # 协议 API
  RegionCascade.tsx   # 标准 React 受控组件（value / onChange，不依赖 host）
  useRegionBinding.ts # A2UI 绑定：path 读写、dispatchA2UIAction
  element.tsx         # 薄桥接：ensureComponentStyles + createReactComponent
  regionData.ts       # 业务数据（可选）
  index.ts
  index.less
```

| 文件 | 职责 |
|------|------|
| `RegionCascade.tsx` | 普通 React 组件，只认 `value`、`onChange` 等 props |
| `useRegionBinding.ts` | `resolveBoundValue` / `writeBoundValue` / 动作派发 |
| `element.tsx` | 仅注册 CustomElement，**不在 createReactComponent 回调里写 Hooks** |

### API 定义规范

```ts
import { z } from 'zod';
import { defineComponentApi, DynString } from '@bote/a2ui-custom-kit';

export const RemoteReactBadgeApi = defineComponentApi({
  name: 'RemoteReactBadge',
  shape: {
    title: DynString,
    subtitle: DynString.optional(),
    tone: z.enum(['purple', 'blue']).optional(),
  },
});

```

规则：

1. 需要 Agent 绑定或 `path` 的字段用 **`DynString`**（或 `web_core` 的 `CommonSchemas`）。
2. 需要由 Agent 配置点击、提交等交互时，增加 **`action: ActionSchema.optional()`**，组件内用 **`dispatchDeclaredAction(host)`**（见「交互」章节）。
3. `name` 与协议里 `"component": "BizPromoCard"` **完全一致**。
4. schema 用 **`.strict()`**（`defineComponentApi` 已内置）。
5. 注册时通过 `defineRegistryEntry` 自动附带 JSON Schema。

参考： https://a2ui-composer.ag-ui.com/custom-catalog/catalog-components

### element.tsx 开发模式（二选一）

#### 模式 A：原生 HTMLElement

使用 `createNativeElement` + runtime 工具：

```ts

import {
  createNativeElement,
  dispatchDeclaredAction,
  ensureComponentStyles,
  readComponentProps,
  readStringProp,
} from '@bote/a2ui-custom-kit';
import styles from './index.less';

const STYLE_KEY = 'remote-glow-capsule';
const PREFIX = 'remote-glow-capsule';


export const RemoteGlowCapsuleElement = createNativeElement('RemoteGlowCapsuleElement', {
  render(host) {
    // 渲染组件样式
    ensureComponentStyles(host, STYLE_KEY, styles);
    // 读取host.componentProps ,真实运行时时这样调用的
    const props = readComponentProps(host);
    // 获取参数值，没有获取到以最后一个参数为默认值
    const label = readStringProp(props, 'label', 'Remote');
    const tone = readStringProp(props, 'tone', 'purple');

    host.replaceChildren();
    host.className = PREFIX;
    // 创建一个原声的html元素，挂载到host 上
    const wrap = document.createElement('button');
     wrap.className = `${PREFIX}__btn ${toneClass}`;
    wrap.type = 'button';
    wrap.textContent = label;
    wrap.onclick = () => {
      // 按协议 props.action 派发（见下文「协议声明式 action」）
      dispatchDeclaredAction(host);
    };
    host.appendChild(wrap);
  },
});

```

#### 模式 B：React 桥接（推荐：复杂 UI）

使用 `createReactComponent`：

```tsx
import React from 'react';
import { createReactComponent, dispatchA2UIAction, ensureComponentStyles } from '@bote/a2ui-custom-kit';
import { RemoteReactBadgeApi } from './api';
import styles from './index.less';

const STYLE_KEY = 'remote-react-badge';
const PREFIX = 'remote-react-badge';

// createReactComponent 第一个参数需要该组件的api定义
export const RemoteReactBadgeElement = createReactComponent(RemoteReactBadgeApi, ({ props, host }) => {
  // 挂在组件样式
  ensureComponentStyles(host, STYLE_KEY, styles);

  // 下面就是和react组件开发过程一样
  const title = String(props.title ?? 'React Remote');
  const subtitle = props.subtitle != null ? String(props.subtitle) : '';
  const tone = props.tone === 'blue' ? 'blue' : 'purple';
  const cardClass = tone === 'blue' ? `${PREFIX}__card ${PREFIX}__card--tone-blue` : `${PREFIX}__card ${PREFIX}__card--tone-purple`;

  return (
    <div className={PREFIX}>
      <button
        type="button"
        className={cardClass}
        onClick={() => {
          // 派发事件
          dispatchA2UIAction(host, {
            name: 'remote_react_badge_click',
            context: { source: 'RemoteReactBadge', title, subtitle, tone },
          });
        }}
      >
        <p className={`${PREFIX}__title`}>{title}</p>
        {subtitle ? <p className={`${PREFIX}__subtitle`}>{subtitle}</p> : null}
      </button>
    </div>
  );
});
```

**适用**：简单展示、按钮、卡片等逻辑较少的组件。

#### 模式 B+：标准 React 分层（推荐：antd、表单、级联）

`createReactComponent` 的 **render 回调不是 React 函数组件**，不能在回调里直接使用 `useState` / `useEffect`（会触发 React #321）。应拆成：

1. **`Xxx.tsx`**：标准受控组件  
2. **`useXxxBinding.ts`**：把 `host` + 协议 `props` 转成受控 props  
3. **`element.tsx`**：只 `ensureComponentStyles` 并 `return <XxxHost />`

**`RegionCascade.tsx`（与 A2UI 无关的 UI）**

```tsx
import React, { useCallback, useRef } from 'react';
import { ConfigProvider, Select } from 'antd';

export type RegionCascadeValue = {
  province?: string;
  city?: string;
  county?: string;
};

export type RegionCascadeProps = {
  value?: RegionCascadeValue;
  onChange?: (value: RegionCascadeValue) => void;
  disabled?: boolean;
  provincePlaceholder?: string;
  cityPlaceholder?: string;
  countyPlaceholder?: string;
};

export function RegionCascade({ value, onChange, disabled, ...placeholders }: RegionCascadeProps) {
  const popupMountRef = useRef<HTMLDivElement>(null);
  const { province, city, county } = value ?? {};

  const getPopupContainer = useCallback(
    () => popupMountRef.current ?? document.body,
    [],
  );

  return (
    <ConfigProvider getPopupContainer={getPopupContainer}>
      <div className="remote-region-cascade">
        <Select
          value={province}
          onChange={(v) => onChange?.({ province: v, city: undefined, county: undefined })}
          {/* ... 市、县 Select */}
        />
        <div ref={popupMountRef} className="remote-region-cascade__popup-mount" />
      </div>
    </ConfigProvider>
  );
}
```

**`useRegionBinding.ts`（A2UI 数据与动作）**

```ts
import { useCallback, useEffect, useMemo, useState } from 'react';
import {
  dispatchA2UIAction,
  resolveBoundValue,
  writeBoundValue,
  type A2UICustomElementHost,
} from '@bote/a2ui-custom-kit';
import type { RegionCascadeProps, RegionCascadeValue } from './RegionCascade';

function readRegionCode(host: A2UICustomElementHost, raw: unknown): string | undefined {
  if (typeof raw === 'string') return raw || undefined;
  return resolveBoundValue(host, raw) || undefined;
}

export function useRegionBinding(host: A2UICustomElementHost, apiProps: ApiProps): RegionCascadeProps {
  const fromEngine = useMemo(
    () => ({
      province: readRegionCode(host, apiProps.province),
      city: readRegionCode(host, apiProps.city),
      county: readRegionCode(host, apiProps.county),
    }),
    [host, apiProps.province, apiProps.city, apiProps.county],
  );

  const [value, setValue] = useState(fromEngine);
  useEffect(() => setValue(fromEngine), [fromEngine]);

  const onChange = useCallback(
    (next: RegionCascadeValue) => {
      setValue(next);
      writeBoundValue(host, apiProps.province, next.province ?? '');
      writeBoundValue(host, apiProps.city, next.city ?? '');
      writeBoundValue(host, apiProps.county, next.county ?? '');
      dispatchA2UIAction(host, { name: 'remote_region_change', context: { /* ... */ } });
    },
    [host, apiProps.province, apiProps.city, apiProps.county],
  );

  return { value, onChange, disabled: Boolean(apiProps.disabled) };
}
```

**`element.tsx`（薄桥接）**

```tsx
import React from 'react';
import { createReactComponent, ensureComponentStyles } from '@bote/a2ui-custom-kit';
import { RemoteRegionCascadeApi } from './api';
import { RegionCascade } from './RegionCascade';
import { useRegionBinding } from './useRegionBinding';
import styles from './index.less';

function RegionCascadeHost({ host, apiProps }) {
  const cascadeProps = useRegionBinding(host, apiProps);
  return <RegionCascade {...cascadeProps} />;
}

export const RemoteRegionCascadeElement = createReactComponent(
  RemoteRegionCascadeApi,
  ({ props, host }) => {
    ensureComponentStyles(host, 'remote-region-cascade', styles);
    return <RegionCascadeHost host={host} apiProps={props} />;
  },
);
```

**要点**

- 需要 Agent / 数据模型绑定的字段在 `api.ts` 里用 **`DynString`**；`useRegionBinding` 里用 `resolveBoundValue` / `writeBoundValue`。
- 使用 **antd** 时：组件内 **不必** 再打包注入 `antd.css`（见下文「渲染」`injectAntdStylesInShadow`）；`Select` 建议 `getPopupContainer` 指向 Shadow 内节点（如 `__popup-mount`）。
- 省市区等静态数据可单独 `regionData.ts`；`element-china-area-data` v6 编码示例：北京 `11` / `1101` / `110101`（勿用旧版 `110000`）。

## Action交互：数据写入与 action 派发

相关 API 均从 `@bote/a2ui-custom-kit` 导出。`host` 为 `createReactComponent` / `createNativeElement` 回调里的自定义元素实例，已挂载 v0.9 的 `context`。

### 总览

| API | 作用对象 | 典型用途 |
|-----|----------|----------|
| **`writeBoundValue`** | Surface **DataModel**（数据模型） | 表单改值、级联清空、与协议 `path` 双向绑定 |
| **`dispatchDeclaredAction`** | 读取协议 **`props.action`** 后进入 `onAction` | 可点击组件与官方 **Button** 一致：动作名与 context 由 Agent 在 JSON 里声明 |
| **`dispatchA2UIAction`** | 页面 **`BaseRenderer` 的 `onAction`** | 组件内**手写**动作名与 context，不依赖协议 `action` 字段 |

**数据与动作分工**：改模型用 `writeBoundValue`；通知业务用 `dispatchDeclaredAction` 或 `dispatchA2UIAction`。表单类组件（如 `RemoteRegionCascade`）常在 `onChange` 里**同时** `writeBoundValue` + `dispatchA2UIAction`。

#### 开发过程中如何处理 action

A2UI v0.9 里，**action** 是组件 props 上的可选字段，类型与官方 catalog 的 **`Action`** 一致。远程组件开发时建议按下面顺序选型：

| 场景 | 推荐做法 |
|------|----------|
| 点击、提交等「动作由 Agent 在协议里配置」 | `api.ts` 声明 `action: ActionSchema.optional()`，交互处调用 **`dispatchDeclaredAction(host)`** |
| 动作名固定、或 context 大量依赖运行时 UI 状态 | 继续用 **`dispatchA2UIAction(host, { name, context })`** |
| 既要改 DataModel 又要通知业务 | `writeBoundValue` + 上述二者之一（或组合） |

**不要混用两种派发语义**：若协议里已经写了 `action`，点击时应优先 **`dispatchDeclaredAction`**，避免在组件里再写死另一套 `name`，否则 Agent 改 JSON 不生效。

##### 1. 在 `api.ts` 声明 `action`

```ts
import { z } from 'zod';
import { defineComponentApi, DynString, ActionSchema } from '@bote/a2ui-custom-kit';

export const RemoteGlowCapsuleApi = defineComponentApi({
  name: 'RemoteGlowCapsule',
  shape: {
    label: DynString,
    tone: z.enum(['purple', 'blue']).optional(),
    action: ActionSchema.optional(),
  },
});
```

- **官方 event 包裹**（与内置 Button 一致）：

```json
{
  "component": "RemoteGlowCapsule",
  "label": "点击胶囊",
  "tone": "purple",
  "action": {
    "event": {
      "name": "remote_glow_click",
      "context": {
        "source": "RemoteGlowCapsule",
        "label": { "path": "/demo/label" }
      }
    }
  }
}
```

- **扁平 `name` + `context`**（引擎归一化后同样可读）：

```json
{
  "action": {
    "name": "remote_glow_click",
    "context": { "source": "RemoteGlowCapsule" }
  }
}
```

`context` 里的值可以是字面量，也可以是 **`{ "path": "/xxx" }`**；派发前会经 `dataContext.resolveDynamicValue` 解析（与官方 Button 行为一致）。

> **注意**：`Action` 还支持 **`functionCall`** 形态。当前 **`dispatchDeclaredAction` 只处理带 `event` 或扁平 `name` 的声明**，不包含 `functionCall`。若协议使用 `functionCall`，需在组件内自行读取 `props.action` 并处理，或改用 `dispatchA2UIAction`。

#### 2. 在 `element` 里调用 `dispatchDeclaredAction`

原生与 React 桥接写法相同：在点击、提交等回调里**只传 `host`**，动作内容来自 `readComponentProps(host).action`。

```ts
import {
  createNativeElement,
  dispatchDeclaredAction,
  ensureComponentStyles,
  readComponentProps,
  readStringProp,
} from '@bote/a2ui-custom-kit';

wrap.onclick = () => {
  dispatchDeclaredAction(host);
};
```

```tsx
<button type="button" onClick={() => dispatchDeclaredAction(host)}>
  {label}
</button>
```

若 `props.action` 缺失或格式无法识别，**`dispatchDeclaredAction` 静默返回**（不抛错、不派发），便于可选 action 的组件。

##### 手写 `dispatchA2UIAction` 的对比

| 维度 | `dispatchDeclaredAction` | `dispatchA2UIAction` |
|------|--------------------------|------------------------|
| 动作名与 context 来源 | 协议 JSON 的 **`action`** | 组件代码里传入的 **`detail`** |
| Agent 是否可改动作 | 是，改协议即可 | 否，需改组件 bundle |
| 与官方 Button | 一致（`context` 内 path 会解析） | 远程扁平 `{ name, context }` |
| 无 `action` 时 | 静默不派发 | 由你是否调用决定 |


##### 页面侧 `onAction`

无论走 `dispatchAction` 还是回退的 `a2uiaction`，宿主应用最终在 **`onAction`** 里收到的都是扁平结构：

```tsx
<BaseRenderer
  onAction={({ name, context }) => {
    if (name === 'remote_glow_click') {
      // context 已解析 path，可直接使用
    }
  }}
/>
```

#### `writeBoundValue`

#### 签名

```ts
import { writeBoundValue, type A2UICustomElementHost } from '@bote/a2ui-custom-kit';

writeBoundValue(
  host: A2UICustomElementHost,
  raw: unknown,    // 协议里该字段的绑定描述，一般用 apiProps.xxx
  value: unknown,  // 要写入 DataModel 的值
): void
```

##### 行为说明

1. 从 `raw` 解析 JSON Pointer：仅识别 **`{ "path": "/region/province" }`**（无 `path` 则直接返回，不抛错）。
2. 调用 `host.context.dataContext.set(path, value)` 更新当前 Surface 的 DataModel。
3. 绑了同 path 的其它组件会联动更新；当前元素 `componentModel.onUpdated` 触发后，`createReactComponent` 会重新 paint。

#### 参数 `raw` 怎么传

| 传法 | 是否有效 |
|------|----------|
| `apiProps.province`（协议为 `{ "path": "/region/province" }`） | 推荐 |
| 手写 `{ path: "/region/province" }` | 可以 |
| 引擎归一化后的字符串 `"11"` | 无效（无 path） |
| 仅 `literalString` 绑定 | 无效 |

需要进 DataModel 时：`api.ts` 用 **`DynString`**，协议 JSON 写 **`path`**，并配合 `updateDataModel` 给初始值。

##### 参数 `value`

类型为 **`unknown`**（字符串、数字、对象等均可）。清空下级时常用 `''`。

#### 配套读取：`resolveBoundValue`

```ts
import { resolveBoundValue } from '@bote/a2ui-custom-kit';

const current = resolveBoundValue(host, apiProps.province);
```

引擎若已将 `componentProps.province` 归一成字符串，也可直接读 `componentProps`；有 path 时以 DataModel 为准更稳妥。

##### 示例

```ts
writeBoundValue(host, apiProps.province, '11');
writeBoundValue(host, apiProps.city, '');
writeBoundValue(host, apiProps.county, '');
```

协议片段：

```json
{
  "component": "RemoteRegionCascade",
  "province": { "path": "/region/province" },
  "city": { "path": "/region/city" },
  "county": { "path": "/region/county" }
}
```

```json
{
  "updateDataModel": {
    "surfaceId": "remote_glow_demo",
    "path": "/",
    "value": {
      "region": {
        "province": "11",
        "city": "1101",
        "county": "110101"
      }
    }
  }
}
```

#### `dispatchA2UIAction`

组件内**自行构造**动作名与 context 时使用。若动作已由协议 **`action`** 字段声明，应优先 **`dispatchDeclaredAction`**（见上文）。

##### 行为说明

在自定义元素上派发 **`a2uiaction`** 自定义事件（`bubbles: true`、`composed: true`，可穿出 Shadow DOM）。`@bote/a2ui-render` 的 `LitSurfaceHost` 监听后，经 `mergeLitActionPayload` 交给页面 **`onAction`**。

**不会**调用 `dataContext.set`，**不会**修改 DataModel。

##### `name`

自定义动作名，建议带组件前缀，例如 `remote_region_change`、`remote_glow_click`。页面在 `onAction` 里按 `name` 分支。

##### `context`

附带给业务的载荷，字段自定，例如：

```ts
dispatchA2UIAction(host, {
  name: 'remote_region_change',
  context: {
    source: 'RemoteRegionCascade',
    province: '11',
    city: '1101',
    county: '110101',
    provinceName: '北京市',
    cityName: '市辖区',
    countyName: '东城区',
  },
});
```

页面侧：

```tsx
<BaseRenderer
  messages={messages}
  protocolVersion="0.9"
  customComponents={customComponents}
  onAction={({ name, context }) => {
    if (name === 'remote_region_change') {
      // 调接口、写 store、埋点等
    }
    if (name === 'remote_glow_click') {
      // ...
    }
  }}
/>
```

远程自定义组件使用扁平结构 `{ name, context }` 即可；渲染器会原样规范化 `context` 后回调（与内置 `Button` 的官方 `detail.action` 结构不同）。

#### 推荐组合（表单 + 通知业务）

```ts
const onChange = (next: RegionCascadeValue) => {
  setValue(next);

  writeBoundValue(host, apiProps.province, next.province ?? '');
  writeBoundValue(host, apiProps.city, next.city ?? '');
  writeBoundValue(host, apiProps.county, next.county ?? '');

  dispatchA2UIAction(host, {
    name: 'remote_region_change',
    context: {
      source: 'RemoteRegionCascade',
      ...next,
      provinceName: getRegionLabel(regionTree, next.province ?? ''),
      // ...
    },
  });
};
```

完整实现见 `RemoteRegionCascade/useRegionBinding.ts`。

#### 快速对照

| 需求 | 使用 |
|------|------|
| 把 `/region/city` 更新为 `1101` | `writeBoundValue` |
| 让 Surface 内其它绑 path 的组件联动 | `writeBoundValue` |
| 点击动作由 Agent 在 JSON 里配置 | `action` + `dispatchDeclaredAction` |
| 通知聊天页「用户改了地区」（context 运行时组装） | `dispatchA2UIAction` |
| 只有 `literalString`、要进模型 | 改协议为 `path` 后再 `writeBoundValue` |
| 纯按钮、动作写死在组件里 | 仅 `dispatchA2UIAction` |
| 纯按钮、动作名与 context 跟协议走 | `action` + `dispatchDeclaredAction` |

## 渲染

博特的渲染引擎会默认读取平台上所有上传的自定义组件，如果需要自己去做渲染，平台也提供了 @bote/a2ui-render做渲染


### 远程自定义组件怎么接到渲染上

**单个远程地址**

```ts
import {
  loadRemoteA2UICustomRegistry,
  mergeRegistryEntries,
} from '@bote/a2ui-render';
import type { A2UICustomComponentRegistry } from '@bote/types';

const remotePart = await loadRemoteA2UICustomRegistry(
  'https://cdn.example.com/a2ui-remote/RemoteGlowCapsule.mjs',
);

const customComponents: A2UICustomComponentRegistry = mergeRegistryEntries(
  staticRegistry,
  remotePart,
);
```

**多个远程地址一次合并**

```ts
import {
  loadRemoteA2UICustomRegistries,
  mergeRegistryEntries,
} from '@bote/a2ui-render';
import type { A2UICustomComponentRegistry } from '@bote/types';

const remotePart = await loadRemoteA2UICustomRegistries([
  'https://cdn.example.com/a2ui-remote/RemoteGlowCapsule.mjs',
  'https://cdn.example.com/a2ui-remote/RemoteReactBadge.mjs',
]);

const customComponents: A2UICustomComponentRegistry = mergeRegistryEntries(
  staticRegistry,
  remotePart,
);
```

**渲染**：合并得到的 **`customComponents`** 直接交给 **`BaseRenderer`**（在 **`useEffect`**、路由 **`loader`** 等能 **`await`** 的地方先加载好，再放进 **`useState` 或 store** 即可）。

```tsx
<BaseRenderer
  messages={messages}
  protocolVersion="0.9"
  customComponents={customComponents}
  onAction={handleAction}
/>
```

### 使用 antd 的远程组件：Shadow 内样式

A2UI 预览在 **`a2ui-surface` 及内置组件的嵌套 ShadowRoot** 中渲染，全局 `antd.css` **不会自动穿透**。`@bote/a2ui-render` 提供开关 **`injectAntdStylesInShadow`**：

- 构建时内联一份 `antd.min.css`，运行时解析为 **共享 `CSSStyleSheet`**，挂到各层 Shadow 的 `adoptedStyleSheets`（CSS 只解析一次）。
- 远程组件 **无需** 在 bundle 里 `import 'antd/dist/antd.min.css'`，也不必在组件里 `ensureAntdStyles`。

```tsx
<BaseRenderer
  messages={messages}
  protocolVersion="0.9"
  customComponents={customComponents}
  injectAntdStylesInShadow
  onAction={handleAction}
/>
```

本地联调可参考 `bote-web/chatBot` 的 `A2UITest`：示例 `remote` 已开启该开关。远程组件仍须处理 **下拉层挂载**（`ConfigProvider` + `getPopupContainer` 指向 Shadow 内容器）。

协议示例（省市区 path 绑定 + 初始数据）见 `chatBot/src/pages/A2UITest/mock/v0.9/demo-remote-only.json`。
